# iEtermAI 操作文档

本文档记录当前已经实现并测试过的 Mac 版 iEterm 自动化流程。当前目标是自动完成登录前准备，并支持查询类指令；不做订座、出票、退票、废票等高风险操作。

## 1. 准备环境

进入项目目录：

```bash
cd /Users/cursor_workspace/iEtermAI
```

如果还没有虚拟环境，先创建并安装依赖：

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

如果已经创建过虚拟环境，只需要激活：

```bash
source .venv/bin/activate
```

## 2. Mac 权限设置

第一次使用真实 iEterm 自动化前，需要开启辅助功能权限：

1. 打开 Mac 的 `系统设置`
2. 进入 `隐私与安全性`
3. 打开 `辅助功能`
4. 允许 `Cursor`、`Terminal` 或实际运行服务的程序控制电脑

如果没有授权，可能会出现无法读取窗口、无法点击按钮、无法选择线路等问题。

## 3. 启动服务

真实操作 Mac 版 iEterm 时使用 `macos` 后端：

```bash
IETERM_AUTOMATION_BACKEND=macos uvicorn app.main:app --reload
```

如果只是测试接口结构，不操作真实 iEterm，可以使用 `mock`：

```bash
IETERM_AUTOMATION_BACKEND=mock uvicorn app.main:app --reload
```

## 4. 打开 iEterm 并读取线路

确保 iEterm 处于登录弹窗页面。服务会尝试自动打开或激活 `iEterm Mac版`。

读取可选线路：

```bash
curl http://127.0.0.1:8000/session/login-aliases
```

返回示例：

```json
{
  "aliases": ["can826-01", "BJS177", "can826-05", "can826-06", "584-02", "其他"],
  "selected_alias": "can826-01"
}
```

## 5. 选择线路

例如选择 `can826-06`：

```bash
curl -X POST http://127.0.0.1:8000/session/login-alias \
  -H "Content-Type: application/json" \
  -d '{"alias": "can826-06"}'
```

返回里的 `selected_alias` 应该变成 `can826-06`。

## 6. 登录

登录前请先确认 iEterm 弹窗里的用户名、密码、认证方式、服务器、端口都是正确的。系统不会读取或保存你的密码，只会点击 `登 录` 按钮。

执行登录：

```bash
curl -X POST http://127.0.0.1:8000/session/login
```

登录成功后，iEterm 会出现“系统提示”弹窗，内容类似：

```text
系统提示
尊敬的 m25123006
欢迎使用广州美亚eterm
确定
```

自动化流程会继续点击这个弹窗里的 `确定` 按钮。只有点掉这个弹窗后，才算真正进入可以查票的状态。

## 7. 检查登录状态

```bash
curl http://127.0.0.1:8000/session/status
```

如果返回里 `state` 是 `logged_in`，说明当前可继续查询。

## 8. 国际票价查询

当前国际票价查询接口：

```bash
curl -X POST http://127.0.0.1:8000/query/international-fare \
  -H "Content-Type: application/json" \
  -d '{
    "origin": "BJS",
    "destination": "TYO",
    "departure_date": "2026-06-01",
    "airline": "CA",
    "passenger_type": "ADT"
  }'
```

默认生成的指令类似：

```text
XS FSD BJSTYO/CA
```

不同 iEterm 环境的国际票价指令可能不同。如果真实系统不是这个格式，需要修改 `.env` 里的：

```bash
IETERM_INTERNATIONAL_FARE_COMMAND_TEMPLATE=XS FSD {origin}{destination}{airline_part}
```

## 9. 完整流程速查

```bash

source .venv/bin/activate
IETERM_AUTOMATION_BACKEND=macos uvicorn app.main:app --reload
```

另开一个终端：

```bash

curl -X POST http://127.0.0.1:8000/session/login-alias \

curl http://127.0.0.1:8000/session/status
```

## 10. 常见问题

### 读取不到线路

确认 iEterm 是否停留在登录弹窗，且 Mac 辅助功能权限已打开。

### 登录后还不能查票

确认“系统提示”弹窗是否已经点掉。弹窗内容通常是欢迎使用广州美亚 eterm，必须点击 `确定`。

### 返回 `logged_in` 但界面还在登录页

这是登录状态识别问题，需要根据当前 iEterm 控件结构继续增强检测。当前代码已经加入对 `登 录` 按钮和系统提示弹窗的识别。

### 查价结果解析不准确

先保留 `raw_text` 原始黑屏文本，再根据真实返回格式调整 `app/parser.py`。第一版解析器只会保守识别明确包含币种和金额的行。
