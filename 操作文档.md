# iEtermAI 操作文档

本文档记录当前已经实现并测试过的 Mac 版 iEterm 自动化流程。当前目标是自动完成登录前准备，并支持从手机发送 iEterm 指令。

## 1. 准备环境

进入项目目录：

```bash
cd /Users/cursor_workspace/iEtermAI
```

如果还没有虚拟环境，先创建并安装依赖：

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

如果已经创建过虚拟环境，只需要激活：

```bash
source .venv/bin/activate
```

## 2. Mac 权限设置

第一次使用真实 iEterm 自动化前，需要开启辅助功能权限：

1. 打开 Mac 的 `系统设置`
2. 进入 `隐私与安全性`
3. 打开 `辅助功能`
4. 允许 `Cursor`、`Terminal` 或实际运行服务的程序控制电脑

如果没有授权，可能会出现无法读取窗口、无法点击按钮、无法选择线路等问题。

如果要在手机页面查看当前 `iEterm` 截图，还需要开启屏幕录制权限：

1. 打开 Mac 的 `系统设置`
2. 进入 `隐私与安全性`
3. 打开 `屏幕录制`
4. 允许 `Cursor`、`Terminal` 或实际运行服务的程序录制屏幕

如果没有开启，手机页面里的截图可能是空白、黑屏，或者无法更新。

## 3. 启动服务

真实操作 Mac 版 iEterm 时使用 `macos` 后端：

```bash
IETERM_AUTOMATION_BACKEND=macos uvicorn app.main:app --reload
```

如果要让手机通过 Tailscale 访问，需要监听所有网卡：

```bash
IETERM_AUTOMATION_BACKEND=macos uvicorn app.main:app --host 0.0.0.0 --port 8000
```

如果只是测试接口结构，不操作真实 iEterm，可以使用 `mock`：

```bash
IETERM_AUTOMATION_BACKEND=mock uvicorn app.main:app --reload
```

## 3.1 手机远程访问设置

第一版推荐使用免费的 Tailscale：

1. Mac 安装并登录 Tailscale
2. 手机安装并登录同一个 Tailscale 账号
3. 在 Mac 的 Tailscale 菜单中查看本机 IP，通常是 `100.x.x.x`
4. Mac 启动服务时使用 `--host 0.0.0.0 --port 8000`
5. 手机浏览器打开：

```text
http://Mac的Tailscale-IP:8000/mobile
```

例如：

```text
http://100.64.12.34:8000/mobile
```

建议在 `.env` 中设置手机访问 Token：

```bash
IETERM_MOBILE_ACCESS_TOKEN=换成你自己的密码
```

手机页面首次打开时，把这个 Token 输入到页面顶部并保存。之后页面会自动带着 Token 调用接口。

## 4. 打开 iEterm 并读取线路

确保 iEterm 处于登录弹窗页面。服务会尝试自动打开或激活 `iEterm Mac版`。

读取可选线路：

```bash
curl http://127.0.0.1:8000/session/login-aliases
```

返回示例：

```json
{
  "aliases": ["can826-01", "BJS177", "can826-05", "can826-06", "584-02", "其他"],
  "selected_alias": "can826-01"
}
```

## 5. 选择线路

例如选择 `can826-06`：

```bash
curl -X POST http://127.0.0.1:8000/session/login-alias \
  -H "Content-Type: application/json" \
  -d '{"alias": "can826-06"}'
```

返回里的 `selected_alias` 应该变成 `can826-06`。

## 6. 登录

登录前请先确认 iEterm 弹窗里的用户名、密码、认证方式、服务器、端口都是正确的。系统不会读取或保存你的密码，只会点击 `登 录` 按钮。

执行登录：

```bash
curl -X POST http://127.0.0.1:8000/session/login
```

登录成功后，iEterm 会出现“系统提示”弹窗，内容类似：

```text
系统提示
尊敬的 m25123006
欢迎使用广州美亚eterm
确定
```

自动化流程会继续点击这个弹窗里的 `确定` 按钮。它会先找标准按钮；如果 iEterm 把按钮画在黑屏区域里，程序会截图识别蓝色 `确定` 按钮后再点击。

点掉弹窗后，程序还会点击左上方黑色输入区域，并按一次 `ESC`。这一步是为了让黑屏里出现绿色输入箭头。只有绿色箭头出现后，才算真正进入可以查票、可以发送指令的状态。

## 7. 检查登录状态

```bash
curl http://127.0.0.1:8000/session/status
```

如果返回里 `state` 是 `logged_in`，说明当前可继续查询。

## 7.1 基础查票指令

当前实际查票可以使用 `AVH/城市对日期/航司` 格式。

示例：

```text
AVH/CANHRE30JUN/ADD
```

含义：

- `AVH`：航班可用性查询
- `CANHRE`：广州到哈拉雷，前三位是出发城市/机场，后三位是到达城市/机场
- `30JUN`：出发日期，30 日 6 月
- `ADD`：中转点或航司/线路参数，按 iEterm 实际规则填写

黑屏返回中常见字段：

- 航班号：例如 `ET607`、`ET873`
- 起飞时间：例如 `0030`
- 落地时间：例如 `0540`
- `+1`：第二天到达
- `0`：没有经停
- `1`：有经停
- `Y7 G7 S7 B7 M7`：不同舱位的余位，数字越大表示余位越多

如果手机表单暂时没有覆盖某种复杂格式，可以直接在“原始查询指令”里输入完整指令，例如：

```text
AVH/CANHRE30JUN/ADD
```

手机“原始查询指令”现在不再限制指令类型，可以发送 `AVH`、`SD1L1` 等任意 iEterm 指令。发送指令后，程序会自动按 Mac 的 `Enter` 键，也就是键盘上的 `fn + enter`，用于触发 iEterm 查询。

如果已经走到票价计算步骤，手机页面可以点击 `票价计算并复制弹窗`。程序会点击 iEterm 里的 `票价计算` 按钮，等弹窗出现后点击 `复制内容`，并把弹窗内容原样显示在手机结果区。

## 8. 国际票价查询

当前国际票价查询接口：

```bash
curl -X POST http://127.0.0.1:8000/query/international-fare \
  -H "Content-Type: application/json" \
  -d '{
    "origin": "BJS",
    "destination": "TYO",
    "departure_date": "2026-06-01",
    "airline": "CA",
    "passenger_type": "ADT"
  }'
```

默认生成的指令类似：

```text
XS FSD BJSTYO/CA
```

不同 iEterm 环境的国际票价指令可能不同。如果真实系统不是这个格式，需要修改 `.env` 里的：

```bash
IETERM_INTERNATIONAL_FARE_COMMAND_TEMPLATE=XS FSD {origin}{destination}{airline_part}
```

## 8.1 手机网页控制台

手机网页地址：

```text
http://Mac的Tailscale-IP:8000/mobile
```

页面包含这些功能：

- 保存访问 Token
- 查看登录状态
- 读取线路列表
- 选择线路
- 登录并确认系统提示弹窗
- 关闭电脑上的 iEterm，并自动确认退出弹窗
- 用表单查询国际票价
- 输入原始查询指令
- 点击票价计算并复制弹窗内容
- 每次操作完成后自动刷新当前 iEterm 截图
- 查看返回原文和解析结果

页面执行操作时会显示黄色 loading 提示，并临时禁用按钮。看到 loading 时请等待当前操作完成，不要重复点击。

截图区域会显示电脑当前屏幕。点击截图可以打开全屏大图，方便看清黑屏返回内容；点背景或 `关闭` 按钮可以退出大图。建议把 `iEterm Mac版` 窗口撑满并保持在最前面，这样手机上看到的就是 iEterm 当前状态，方便判断指令是否报错、是否需要继续输入下一条指令。

原始指令输入框不再限制指令类型。请确认指令正确后再发送。每次发送过的指令会自动保存到手机浏览器本地的“历史指令”里，最多保留最近 30 条；以后点击某条历史指令，会直接再次发送这条指令。也可以点击 `清空历史指令` 删除本机保存的历史。

## 9. 完整流程速查

```bash
cd /Users/cursor_workspace/iEtermAI
source .venv/bin/activate
IETERM_AUTOMATION_BACKEND=macos uvicorn app.main:app --host 0.0.0.0 --port 8000
```

另开一个终端：

```bash
curl http://127.0.0.1:8000/session/login-aliases

curl -X POST http://127.0.0.1:8000/session/login-alias \
  -H "Content-Type: application/json" \
  -d '{"alias": "can826-06"}'

curl -X POST http://127.0.0.1:8000/session/login

curl http://127.0.0.1:8000/session/status

curl -X POST http://127.0.0.1:8000/session/close
```

手机远程使用时，把 `127.0.0.1` 换成 Mac 的 Tailscale IP。

## 9.1 关闭 iEterm

如果要从手机或接口关闭电脑上的 `iEterm Mac版`，调用：

```bash
curl -X POST http://127.0.0.1:8000/session/close
```

当 iEterm 弹出“关闭软件将会造成工作中断，是否确认退出软件？”时，程序会自动点击 `是`。手机页面里也有红色的 `关闭 iEterm` 按钮，点击前会先弹出手机浏览器自己的确认框，避免误触。

## 10. 常见问题

### 读取不到线路

确认 iEterm 是否停留在登录弹窗，且 Mac 辅助功能权限已打开。

### 登录后还不能查票

确认“系统提示”弹窗是否已经点掉。弹窗内容通常是欢迎使用广州美亚 eterm，必须点击 `确定`。测试自动登录时请让 `iEterm Mac版` 保持在最前面，不要让微信、浏览器或其他窗口盖住确认弹窗。

### 出现“用户异常，请重新登录”

如果黑屏返回 `#用户异常,请重新登录!!!`，系统会把当前状态标记为 `session_expired`，本次查询会返回失败。此时在手机页面重新点击 `登录并确认系统提示`，程序会先关闭并重新打开 iEterm，再走登录流程。

### 返回 `logged_in` 但界面还在登录页

这是登录状态识别问题，需要根据当前 iEterm 控件结构继续增强检测。当前代码已经加入对 `登 录` 按钮和系统提示弹窗的识别。

### 查价结果解析不准确

先保留 `raw_text` 原始黑屏文本，再根据真实返回格式调整 `app/parser.py`。第一版解析器只会保守识别明确包含币种和金额的行。

### 手机打不开页面

确认 Mac 和手机都已登录 Tailscale，Mac 服务启动命令里有 `--host 0.0.0.0`，并且手机访问的是 Mac 的 Tailscale IP，不是 `127.0.0.1`。
